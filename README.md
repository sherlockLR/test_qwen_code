# 传记写作助手微信小程序 PRD

## 1. 项目信息
- 项目名称: biography_writing_assistant
- 编程语言: 微信小程序(前端)、Rust(后端)
- LLM模型: QWen3
- 原始需求: 开发一款协助书写传记（自传、他人传记）的微信小程序，后端服务使用Rust实现；LLM的调用使用当前火热的 QWen

## 2. 产品定义

### 2.1 产品目标
1. 为用户提供智能化的传记写作辅助工具，降低传记写作门槛
2. 通过AI技术辅助用户构建完整的传记框架和内容
3. 打造便捷的移动端传记创作与管理平台

### 2.2 用户故事
1. 作为一个想要记录父母人生故事的子女，我希望通过引导式提问采访框架，能够系统地记录下父母的人生经历
2. 作为一个准备写自传的企业家，我想要通过AI辅助，帮助我梳理人生重要节点并生成合适的写作框架
3. 作为一个传记作家，我需要一个便捷的工具来管理多个传记项目，并能够随时记录灵感和素材
4. 作为一个普通用户，我希望能通过AI引导，轻松完成家庭成员的口述史记录


## 3. 技术规格

### 3.1 需求分析
1. 前端技术架构
   - 基于微信小程序开发框架
   - 使用Tailwind CSS进行界面设计
   - 支持富文本编辑和多媒体内容
   
2. 后端技术架构
   - 使用Rust语言开发后端服务
   - 集成DeepSeek LLM API
   - 实现安全的数据存储和用户认证

3. AI功能需求
   - 智能写作建议和内容优化
   - 自动生成写作大纲
   - 智能问答和采访提示

### 3.2 需求池
#### P0（必须实现）
1. 用户账号系统
   - 微信授权登录
   - 个人资料管理
   - 作品管理

2. 核心写作功能
   - 创建新传记项目
   - 基础文本编辑
   - 传记框架模板
   - 写作进度管理

3. AI辅助功能
   - QWen API集成
   - 智能写作建议
   - 采访问题生成

#### P1（应该实现）
1. 增强写作功能
   - 多媒体内容支持
   - 时间线展示
   - 章节管理
   - 版本历史

2. AI高级功能
   - 情感分析
   - 风格建议
   - 智能润色

3. 数据管理
   - 云端存储
   - 本地缓存
   - 数据导出


#### P2（可选实现）
1. 社交功能
   - 作品分享
   - 协作写作
   - 读者反馈

2. 高级特性
   - 语音转文字
   - OCR图片识别
   - 自定义主题
